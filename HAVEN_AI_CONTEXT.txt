WHAT HAVEN IS
━━━━━━━━━━━━━
HAVEN is a civic-tech platform for housing-insecure and underserved residents in
the San José, CA area (and beyond). It gives residents a single place to:
  - Store and manage documents
  - Apply for government benefits and housing programs
  - Message their caseworker
  - Chat with BB, an AI civic concierge
  - Access a Survival Bible (offline-capable survival knowledge base)

It simultaneously gives caseworkers, agencies, and admins a case management
dashboard, workload tools, agency integrations, and reporting.

HAVEN is NOT a government website. It is a private platform that bridges
residents with government and non-profit agencies through a clean, dignified UX.

WHO USES HAVEN
━━━━━━━━━━━━━━
Four user roles exist on the platform:

RESIDENT
  - The person seeking help (housing, food, health, benefits, crisis support, legal, employment)
  - Can create cases, upload documents, chat with BB, fill form templates, message their caseworker
  - Can download their own case packet PDF for court or agency appointments
  - Demo accounts: resident@haven.demo / james@haven.demo / sarah@haven.demo (password: Demo2026!)

CASEWORKER
  - The social worker or volunteer who manages resident cases
  - Can claim cases, create tasks, send messages, use BB to fill forms on behalf of residents
  - Can export full case packet PDFs for inter-agency handoffs
  - Sees workload heatmap showing team capacity
  - Demo account: caseworker@haven.demo (password: Demo2026!)

ADMIN
  - Platform administrator
  - Manages agency integrations (OAuth Connect/Disconnect), user management, audit log, analytics
  - Can connect HAVEN to live government OAuth endpoints (Login.gov, VA, SSA)
  - Demo account: admin@haven.demo (password: Demo2026!)

ARCHITECT
  - Superuser / platform owner (jawknee.rodriquez@gmail.com)
  - Full system access including configuration and all admin capabilities

BB — THE AI CONCIERGE
━━━━━━━━━━━━━━━━━━━━━
BB is HAVEN's built-in AI assistant. She is the heart of the resident experience.

Personality:
  - Warm, calm, dignity-first
  - Never judgmental, never bureaucratic, never robotic
  - Residents feel RELIEF when BB responds
  - Caseworkers feel like BB saved them an hour of work

BB's capabilities:
  1. Multi-turn context-aware chat (remembers the conversation within a session)
  2. Role-aware responses — resident mode uses plain language; caseworker mode
     uses operational language (case IDs, checklists, deadlines)
  3. Form analysis — paste any HTML form and BB extracts all fields
  4. Intelligent autofill — maps resident profile + case intake data to form fields
     semantically (e.g. "first_name" → resident's first name)
  5. Application tracking — tracks submissions across agencies
  6. Crisis detection — detects keywords like "evicted today", "no food", "suicide",
     "domestic violence" and responds with hotlines + HAVEN actions
  7. Browser control (advanced) — BB can drive a headless Playwright browser to
     navigate and fill real agency web forms on behalf of the resident
  8. Survival Bible expert — knows water purification, shelter, fire-starting,
     first aid, navigation, foraging, knots, and wilderness psychology

BB LLM engine:
  - Primary: local Ollama (llama3.2:3b by default) — FREE, no API credits
  - Fallback: Claude Sonnet (via Emergent key, only if Ollama is unavailable)
  - EMERGENT_LLM_KEY env var left blank = Sonnet completely disabled
  - BB greets users differently by role on session start

Crisis response triggers (auto-detected in any message):
  critical: "suicide", "kill myself", "hurt myself", "domestic violence",
            "being hit", "evicted today", "evicted tomorrow", "homeless tonight",
            "child in danger"
  high:     "end it", "abuse", "no food", "kids hungry"
  → BB surfaces the appropriate hotline AND recommends a HAVEN action

CORE FEATURES
━━━━━━━━━━━━━

CASE MANAGEMENT
  - Cases are the central unit of work. Each case belongs to one resident.
  - A caseworker can be assigned (or can self-assign via "Claim").
  - Case statuses: new → enriched → routed → active → resolved → closed
  - Each case has: title, description, category, urgency_score (0–100), intake_data
  - Cases have attached tasks, messages, documents, and form submissions
  - Caseworkers see all cases sorted by urgency_score; residents see only their own
  - Case categories: housing | food | health | benefits | employment | legal | crisis | general

TASKS
  - Created by caseworkers on a case
  - Priority: low | medium | high | urgent
  - Status: open | in_progress | done
  - Due dates optional

DOCUMENTS
  - Residents upload files (ID, income proof, lease, medical records)
  - Types: identity | income | residency | medical | other
  - Caseworkers can mark documents as verified
  - Max 8MB per file
  - Stored as base64 data URLs in MongoDB (demo mode)

MESSAGING
  - Secure per-case messaging between resident and caseworker
  - Messages are threaded per case
  - Read/unread tracking

FORM TEMPLATES (13 pre-built)
  - CalFresh / SNAP (USDA)
  - HUD Section 8 Housing Choice Voucher
  - Medi-Cal / Medicaid (CMS)
  - VA Disability Compensation
  - SSI — Supplemental Security Income (SSA)
  - TANF / CalWORKs Cash Aid (HHS)
  - WIC — Women, Infants & Children
  - Unemployment Insurance / EDD (DOL)
  - IRS VITA Free Tax Filing
  - DMV REAL ID / State ID
  - Eviction Defense Response (Santa Clara County Court)
  - Child Welfare Services Referral (CPS)
  - HAVEN Universal Intake Form
  Each template supports:
  - BB Autofill (one click prefills from user profile + case intake data)
  - Manual completion
  - Submit to backend
  - Download as PDF (HAVEN-branded, ReportLab)

CASE PACKET PDF
  - One-click export: GET /api/cases/{id}/packet.pdf
  - 8-section bundle: cover page, case summary + intake data, people (resident +
    caseworker), all submitted applications, documents index, tasks & notes,
    message digest (last 20), attestation footer
  - Used for: court appearances, appeal packets, inter-agency handoffs
  - Auth: resident owner OR assigned caseworker OR admin → 200; other residents → 403

RESOURCE MAP
  - Interactive map of real San José area resources
  - Types: shelter, food, health, crisis, legal, employment
  - Real locations including: Boccardo Reception Center, Sacred Heart Community
    Service, Valley Homeless Healthcare, Santa Clara County Crisis Line, Bill
    Wilson Center, Law Foundation of Silicon Valley, work2future Career Center,
    Family Supportive Housing, Loaves & Fishes Family Kitchen

AGENCY INTEGRATIONS (12 agencies)
  All agencies can run in simulated mode (deterministic mock responses) OR live
  OAuth mode when credentials are configured.
  
  Agencies:
  - HUD — Section 8 Housing Choice Voucher
  - SSA — Supplemental Security Income
  - USDA — SNAP / CalFresh
  - CMS — Medicaid / Medi-Cal
  - VA — Veterans Benefits Administration
  - IRS — VITA Free Tax Filing
  - DMV — REAL ID / Replacement ID
  - DOL — Unemployment Insurance (EDD)
  - HHS — TANF / CalWORKs
  - CPS — Child Welfare Referral
  - Santa Clara County Superior Court (e-filing)
  - WIC — Women, Infants & Children

  OAuth adapter families:
  - logingov  → Login.gov OIDC + PKCE (used by HUD, USDA, IRS, WIC, DOL, DMV, HHS, CMS)
  - va        → VA Lighthouse API
  - ssa       → SSA developer sandbox
  - oauth2    → generic RFC 6749
  - simulated → default when no credentials configured

  To enable live OAuth for an agency (e.g. HUD_SEC8):
    OAUTH_ENV=sandbox + HUD_SEC8_OAUTH_CLIENT_ID=your_client_id
    (sandbox defaults for authorize/token URLs are built-in)

WORKLOAD HEATMAP
  - Admin/Architect view showing each caseworker's case load scored 0–100
  - Color: emerald (0–49) → amber (50–74) → rose (75–100)
  - Expand any row to see that caseworker's active cases
  - Drag-and-drop case chips between rows to reassign cases

SURVIVAL BIBLE
  - Offline-capable knowledge base covering:
    Water: finding, filtering, boiling, chemical treatment, SODIS
    Food: foraging safety, edible plants (dandelion, chickweed, clover, cattails, acorns)
    Shelter: lean-to, debris hut, A-frame, snow cave, tarp configurations
    Fire: bow drill, tinder, feather sticks, lay patterns
    First aid: bleeding, CPR, burns, hypothermia, fractures, wound cleaning
    Navigation: shadow stick, Polaris, Southern Cross, map/compass
    Weather: cloud reading, lightning (30/30 rule), hypo/hyperthermia
    Tools: 10 essentials, knife maintenance, batoning, cordage
    Knots: bowline, square, clove hitch, taut-line, figure-8, sheet bend,
           trucker's hitch, prusik, double fisherman, alpine butterfly
    Psychology: STOP method, survival priorities, pacing, group dynamics
  - BB knows all of this and can answer step-by-step survival questions
  - Available offline via service worker cache

OFFLINE MODE
  - Service worker caches app shell + API GET responses
  - Strategy: app shell = cache-first; API = network-first with cache fallback
  - Write queue: form submissions while offline go to IndexedDB, sync on reconnect
  - OfflineSyncIndicator shows queue status

ONBOARDING CURRICULUM
  - Role-aware learning modules
  - Quiz with 80% pass threshold to unlock next module
  - Progress tracked per user
  - Sample quiz answers: resident welcome: [1,1,1,0,2]

TECH STACK
━━━━━━━━━━

Frontend:
  - React 19 + TypeScript (Vite build)
  - TailwindCSS + shadcn/ui + Radix primitives
  - Zustand state management (authStore, caseStore, bbChatStore, resourceStore)
  - React Router v7
  - Leaflet + react-leaflet (resource map)
  - IndexedDB offline write queue
  - Web Speech API (voice input/output for BB chat)
  - Service worker for offline read cache

Backend:
  - Python 3.12 + FastAPI + Uvicorn
  - MongoDB (Motor async driver)
  - Pydantic v2 for all data models
  - JWT authentication (python-jose)
  - bcrypt password hashing
  - ReportLab for PDF generation
  - Playwright for BB browser control (optional)
  - httpx for async HTTP (agency adapters, Ollama calls)

AI:
  - Ollama (llama3.2:3b default, local, free)
  - Claude Sonnet 4.5 fallback (optional, requires EMERGENT_LLM_KEY)

Infrastructure (self-hosted):
  - Docker Compose: nginx + frontend + backend + MongoDB + Ollama
  - nginx handles HTTP→HTTPS redirect, /api/* proxy to FastAPI, SPA routing
  - Let's Encrypt SSL (certbot)
  - Domain: homeishaven.cloud (primary), home-is-haven-com redirects to .cloud

API ROUTES (prefix: /api)
━━━━━━━━━━━━━━━━━━━━━━━━━

Auth:
  POST /auth/register          — create account (role: resident|caseworker|admin)
  POST /auth/login             — returns JWT token + user object
  GET  /auth/me                — current user profile

Users:
  GET    /users                — list users (caseworker/admin only, ?role= filter)
  PATCH  /users/me             — update own name/phone

Cases:
  POST   /cases                — create case
  GET    /cases                — list cases (?status= ?category= ?assigned_to_me=)
  GET    /cases/{id}           — case detail + tasks + messages + documents
  PATCH  /cases/{id}           — update case (caseworker/admin)
  POST   /cases/{id}/claim     — caseworker self-assigns
  GET    /cases/{id}/packet.pdf — download 8-section PDF packet

Tasks:
  POST   /tasks                — create task
  GET    /tasks                — list tasks (?case_id= ?status=)
  PATCH  /tasks/{id}           — update task
  DELETE /tasks/{id}           — delete task

Messages:
  POST   /messages             — send message
  GET    /messages?case_id=    — thread for a case

Documents:
  POST   /documents            — upload file (multipart)
  GET    /documents?case_id=   — list documents
  PATCH  /documents/{id}/verify — verify document (caseworker/admin)

BB:
  POST   /bb/chat              — send message to BB, returns {reply, intent, session_id}
  GET    /bb/intro             — role-aware greeting
  GET    /bb/sessions/{id}/history — chat history
  POST   /bb/forms/analyze     — extract fields from HTML form
  POST   /bb/forms/autofill    — map user/case data to form fields
  POST   /bb/applications/track — track an application
  GET    /bb/applications/summary — all tracked applications for current user
  POST   /bb/browser/start     — start Playwright browser session
  POST   /bb/browser/action    — navigate|fill|click|type|submit|screenshot|extract|autofill_all
  WS     /bb/browser/stream/{session_id} — live screenshot stream (requires ?token=JWT)

Form Templates:
  GET    /form-templates                           — list all 13 templates
  GET    /form-templates/{id}                      — full template with fields
  POST   /form-templates/{id}/autofill             — deterministic autofill
  POST   /form-templates/{id}/submit               — submit filled form
  GET    /form-templates/submissions/mine          — my submissions
  GET    /form-templates/submissions/{id}          — single submission
  GET    /form-templates/submissions/{id}/pdf      — download single-form PDF

Agency Integrations:
  GET    /integrations                             — catalog of all agencies
  GET    /integrations/{code}                      — single agency detail
  PATCH  /integrations/{code}/toggle               — connect/disconnect (admin)
  POST   /integrations/submit                      — submit application to agency
  GET    /integrations/submissions                 — all submissions (?case_id= ?integration_code=)
  GET    /integrations/submissions/{id}            — single submission
  POST   /integrations/submissions/{id}/sync       — check status from agency
  GET    /integrations/{code}/oauth/meta           — live vs simulated + adapter_family
  GET    /integrations/{code}/oauth/start          — begin OAuth flow (admin, returns authorize_url)
  GET    /integrations/oauth/callback              — OAuth redirect handler
  POST   /integrations/{code}/oauth/disconnect     — revoke tokens (admin)
  GET    /integrations/_/stats                     — dashboard stats

Resources:
  GET    /resources            — all map resources

Notifications:
  GET    /notifications        — in-app notifications for current user

Analytics:
  GET    /analytics/admin      — system-wide analytics (admin)
  GET    /analytics/workload-heatmap — caseworker load matrix

Health:
  GET    /health               — full health: API + DB + BB (Ollama/fallback) status
  GET    /health/live          — liveness probe
  GET    /health/ready         — readiness probe (DB ping)
